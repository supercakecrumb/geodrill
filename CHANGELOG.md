## v0.4.0 - 2026-07-21
### Added
* A /feedback command files bug reports and feedback into snagbox — send it inline (e.g. "/feedback the flags quiz showed the wrong country") and geodrill files it into its snagbox project via a write-only ingest token. Wired via SNAGBOX_URL and SNAGBOX_INGEST_TOKEN; both are optional, and /feedback replies that feedback is unavailable until they are set.
* Every /train question now carries a "🤷 I don't know" button. Tapping it registers the card as not known (a fail, exactly like a wrong answer — never graded as correct), reveals the correct answer in place (and, for type-your-answer questions, as a follow-up reply since the reveal scrolls out of view), and advances to the next card.
* Type-your-answer language questions (special characters — "which language uses «ñ»?") now offer the same inline @geodriller_bot autocomplete that country questions do — a "⌨️ Type your answer" prefill button plus language-name suggestions in the inline dropdown as you type.
### Changed
* The intro card's middle button is now "🧠 I know" (was "🧠 I know this") — the longer label got truncated to "I k…this" in Telegram's button UI.
* Introductions now round-robin across topics instead of exhausting one topic before starting the next. Previously /study served every item of a topic (dozens of near-identical cards) back-to-back before moving on; the intro candidate order is now tier, then within-topic rank, then topic position, so consecutive introductions rotate across the currently-unlocked topics for variety. No scheduling change — reviews are unaffected.
* The Cities topic now shows a map with a marked location and you type the city's name (previously it asked which country a city was in). Each newly discovered city shows an info card with its country, region, population, elevation, and — for the biggest cities — a short fact. Existing per-city progress is reset so cities are re-introduced biggest-first.
* City maps now render on demand at first send (and are cached as a Telegram file_id thereafter, so each map is rendered at most once), instead of being produced by an offline batch and uploaded ahead of time. There's no manual render/upload step anymore, and Garage object storage became optional persistence rather than a prerequisite — with it configured, on-demand renders are saved back so they survive; without it, a map simply re-renders if its file_id is ever lost.
### Fixed
* Free-text replies sent via Telegram's inline autocomplete in a group chat (e.g. "@geodriller_bot Japan") were graded against the raw text including the bot's own @mention, marking correct answers wrong — this also masqueraded as broken case-insensitive matching. handleText now strips a leading bot @mention before grading.
* On a wrong free-text (type-your-answer) question, the follow-up reply now reveals the correct answer ("❌ Wrong — the answer is Spain") instead of a bare "❌ wrong". The original question message is edited with the answer too, but it scrolls out of view, so the sent reply was leaving the user with no idea what the right answer was. Button questions are unchanged (they already highlight the correct option with ✅).

## v0.3.0 - 2026-07-19
### Added
* GeoGuessr-only mode — a per-user /settings toggle (🌍 GeoGuessr-only, default on) that hides every country without Street View coverage everywhere: study, reviews, /stats counts and accuracy, tier boundaries, MCQ distractors, and autocomplete suggestions. Language topics drop languages not spoken in any covered country (with a conservative "keep what we can't determine" rule). Coverage is precomputed per item at ingest (items.gg_relevant); toggling the setting rebuilds tier gating immediately.

## v0.2.0 - 2026-07-19
### Added
* ModeText exercises whose topic requests autocomplete now show a "⌨️ Type your answer" button that opens Telegram inline mode with typo-tolerant country suggestions; tapping one sends it as a normal message, graded by the existing free-text answer path
* New capitals quiz, both directions as two independently-tracked topics under Countries → Capitals: "What's the capital of 🇨🇴 Colombia?" (type the capital city, with autocomplete capital suggestions) and "🏛 Bogotá is the capital of which country?" (type the country, with the existing autocomplete country suggestions). 242 countries with a capital; multi-capital countries (South Africa, Bolivia, Tanzania, ...) accept every listed capital as a correct answer on the country→capital side, while capital→country only ever asks about each country's primary capital to avoid ambiguity.
* New cities quiz (first slice): "🏙 Which country is Mumbai in?" — the question is a city (451 world cities, biggest to smallest by population), the answer is a country, typed with the existing autocomplete country suggestions. One direction only for now; map/photo recognition and terrain questions stay in the backlog.
* New country → language quiz under Countries → Profiles: "What language do you see on signs in 🇰🇪 Kenya?" (single-choice, 247 countries). Wrong options are drawn only from other countries in the SAME geographic region (19-value region taxonomy), and never include a language that's also spoken in the target country itself — even when a same-region country's primary language happens to be one of the target's own secondary languages (a real, common case with shared colonial languages). main_religion and region are also seeded as facts now (for future sibling quizzes), alongside the multi-valued languages_spoken fact this quiz reads.
* New flags quiz under a Flags root: guess-the-flag shows a flag photo (or an emoji-prefixed text fallback when no asset is ingested yet for that item) and asks for the country by autocomplete-typed name, plus a set-choice mode for 10 visually-confusable flag groups (Chad/Romania, Indonesia/Monaco/Poland, Netherlands/Luxembourg, ...) where a randomly-picked member's flag is shown and you pick which country set it could belong to. 228 single countries (including GB's England/Scotland/Wales subdivisions) plus 10 confusable groups (24 countries) — every seeds/flags.yaml entry accounted for. A new cmd/flagassets command ingests the local flag PNG cache into media_files (sha256 + dimensions, idempotent by local_path); photo sends now cache the Telegram file_id after first upload and reuse it on later sends instead of re-uploading from disk.
* Added a /menu navigation hub with a button for every wired destination (Study, Train, Game, Topics, Stats, Settings, Help); /start now ends there too, and every top-level section plus every Study/Train/Game terminal screen carries a "⬅️ Menu" button back to it, so you can always get from any screen to any other
* Enriched the special-characters (39 -> 51) and common-words (79 -> 124) language quiz datasets with a corpus-audited long tail — Vietnamese horns (ơ ư) and circumflex vowels, Czech ř, Slovak ľ, Russian ы/э, Ukrainian і, Serbian/Macedonian ј, Bulgarian ъ, plus church/school/market/bridge/square/station/open vocabulary across a dozen languages. Every new claim independently clears both corpus-audit checks (frequency floor + cross-language collision)
* /stats now shows your current tier (the gating frontier you're working through), so you can see how far you've unlocked.
* New TLD (country-code top-level domain) quiz, both directions as two independently-tracked topics under Countries → Domains: "Which country uses the domain .de?" (answer a country by typing with autocomplete country suggestions) and "🇩🇪 Germany — which top-level domain?" (type the TLD). 248 countries, including the .uk (not .gb) edge case and famous ones like .tv/.io/.me/.ai.
* Container /topics views now show a "Turn group off"/"Turn group on" button (last row, left of "⬆️ Up"): tapping it enables or disables every quizzable topic in that container's subtree at once, instead of toggling each subtopic by hand
### Changed
* Special characters and common-words quizzes now draw distractors from the same guess-the-language language family (romance, slavic-cyrillic, slavic-latin, nordic, cjk, se-asia, malay-indonesian, indian-scripts — seeds/decks.yaml) instead of merely the same script: script alone was already enough to keep a Cyrillic question off Latin/CJK options, but a Latin-script question could still land on any unrelated Latin-script family (e.g. a Polish item offering a Vietnamese or Icelandic option, since romance/slavic-latin/nordic/se-asia all share the "latin" script). Distractors are now close by language family in both topics.
* Cities are now tiered by their own population (tier 0 = the biggest metros down to tier 6 = ~15k–75k towns) instead of inheriting their country's tier, and the city set is expanded well beyond the original 451 using the open GeoNames cities15000 dataset (CC-BY 4.0) while keeping curated English city names.
* Autocomplete inline suggestions are now scoped to the open question's answer domain — a "capital of X?" question suggests capitals, a country-answer question suggests countries — and the flag emoji has been dropped from suggestion rows.
* Guess-the-language left the FSRS study pipeline for a new /game zone: Language Roulette is now an endless streak run (cross-group distractors easing into same-group ones as your streak grows) with a personal best and run count, not a scheduled review — sentences are no longer introduced or reviewed, and the languages subtree has disappeared from /topics
* Rebuilt /help as a tappable inline-keyboard menu (FSRS, the intro buttons, topics & tiers, commands) instead of one static wall of text framed around the retired language-guessing task
* The /menu hub now lays its destination buttons out in a 2-per-row grid instead of one button per row, so it's compact instead of a tall single column.
* Raised /settings cap ceilings (daily new-skill cap to 500, intro cap to 200) and added -5/+5 buttons for the intro cap alongside the existing +-1 buttons, matching the daily cap's layout
* Simplified /settings: dropped the +-1 buttons from both the daily cap and intro-cap rows (keeping only +-5) and added a "⬅️ Menu" button back to the new navigation hub
* Internal: topics are now driven by a universal descriptor engine (one generic exercise generator + one generic seeder, internal/topics/engine) with a shared country-tier rubric (internal/topics/countrytier); special characters, common words, road sides, and the guess-the-language seed were ported onto it behavior-preserving — no visible change, but new topics (TLDs, country profiles, flags, ...) now land as a descriptor plus a seed dataset instead of duplicated code
* Topic and tier progress row labels no longer include the word "introduced" (now "12/451 · 3 good") so the button text stops overflowing/truncating in Telegram.
* /train's "nothing due" screen now shows real progress (reviews done today, how many reviews are still coming up, how many items are left to learn) and, when a next review is scheduled, explains the time in plain terms ("Your next review unlocks at HH:MM ...") instead of a bare, unexplained "Come back at HH:MM"; when there's truly nothing left at all, it says so plainly instead of printing a stale time
* A wrong answer now shows the correct answer (e.g. "✅ Correct answer: Canberra") instead of only marking the attempt wrong — most useful for typed/autocomplete questions that have no options to reveal it.
### Removed
* Removed the legacy /practice command and all its logic (handler, Stop control, prac: callback prefix, NextPractice/PracticeStatsSince/ListActiveItemsForPractice, help and command-menu mentions): it predates the multi-topic FSRS trainer and is superseded by /train; a future endless-practice mode will be designed separately
### Fixed
* Answering an intro card with "I know this" no longer consumes the daily introduction budget: CountIntroductionsToday now excludes the known outcome (engram.IntroKnown), since that outcome never actually introduces the item
