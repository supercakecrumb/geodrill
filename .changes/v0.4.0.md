## v0.4.0 - 2026-07-21
### Added
* A /feedback command files bug reports and feedback into snagbox — send it inline (e.g. "/feedback the flags quiz showed the wrong country") and geodrill files it into its snagbox project via a write-only ingest token. Wired via SNAGBOX_URL and SNAGBOX_INGEST_TOKEN; both are optional, and /feedback replies that feedback is unavailable until they are set.
* Every /train question now carries a "🤷 I don't know" button. Tapping it registers the card as not known (a fail, exactly like a wrong answer — never graded as correct), reveals the correct answer in place (and, for type-your-answer questions, as a follow-up reply since the reveal scrolls out of view), and advances to the next card.
* Type-your-answer language questions (special characters — "which language uses «ñ»?") now offer the same inline @geodriller_bot autocomplete that country questions do — a "⌨️ Type your answer" prefill button plus language-name suggestions in the inline dropdown as you type.
### Changed
* The intro card's middle button is now "🧠 I know" (was "🧠 I know this") — the longer label got truncated to "I k…this" in Telegram's button UI.
* Introductions now round-robin across topics instead of exhausting one topic before starting the next. Previously /study served every item of a topic (dozens of near-identical cards) back-to-back before moving on; the intro candidate order is now tier, then within-topic rank, then topic position, so consecutive introductions rotate across the currently-unlocked topics for variety. No scheduling change — reviews are unaffected.
* The Cities topic now shows a map with a marked location and you type the city's name (previously it asked which country a city was in). Each newly discovered city shows an info card with its country, region, population, elevation, and — for the biggest cities — a short fact. Existing per-city progress is reset so cities are re-introduced biggest-first.
* City maps now render on demand at first send (and are cached as a Telegram file_id thereafter, so each map is rendered at most once), instead of being produced by an offline batch and uploaded ahead of time. There's no manual render/upload step anymore, and Garage object storage became optional persistence rather than a prerequisite — with it configured, on-demand renders are saved back so they survive; without it, a map simply re-renders if its file_id is ever lost.
### Fixed
* Free-text replies sent via Telegram's inline autocomplete in a group chat (e.g. "@geodriller_bot Japan") were graded against the raw text including the bot's own @mention, marking correct answers wrong — this also masqueraded as broken case-insensitive matching. handleText now strips a leading bot @mention before grading.
* On a wrong free-text (type-your-answer) question, the follow-up reply now reveals the correct answer ("❌ Wrong — the answer is Spain") instead of a bare "❌ wrong". The original question message is edited with the answer too, but it scrolls out of view, so the sent reply was leaving the user with no idea what the right answer was. Button questions are unchanged (they already highlight the correct option with ✅).
